                                                               Linux 操作系统




     什么是操作系统：建立在硬件之上的用来操作硬件设备的软件或程序代码   



     为什么要学习Linux系统？

        1 Linux系统以后用来部署我们的服务器

        2 服务器要求：安全性、稳定性

        3 Linux系统的特性：开源、开放、自由

        4 对Linux系统而言"一切皆文件"



     学习Linux系统需要掌握什么？
    
        1 学习Linux系统下常用工具的使用

        2 常用命令的使用shell（一个软件，可以解析命令让操作系统执行）

        3 理解Linux操作系统的文件结构

        4 如何在Linux系统上部署开发环境或运行环境

   Linux系统只有一个目录 "/" 


   bin   存放可以执行的二进制程序 所有用户都有使用权限

   sbin  有系统管理员权限才能执行的程序

   root  目录是超级管理员的主目录（超级管理员进入的主目录）

   home  存放普通用户的根目录 （普通用户进入的主目录）

   etc   存放系统配置文件

   tmp   存放临时文件 系统会定时清理

   usr   

   lost+found 会把一些应急的文件





   命令
 
   cd    目录切换

    /    根目录

   ls    查看目录下的文件

   ifconfig 或 ip addr


   cd ..返回到上一级

   cd . 到当前目录

   mkdir 创建新目录 

   创建多集目录 

  mkdir -p A/B/C    创建的是在当前目录下创建目录 比如当前目录是root 则A/B/C的真实路径为 /root/A/B/C

  mkdir -p /A/B/C    /A/B/C

  mkdir -p .A/B/C    /root/A/B/C

  mkdir -p ..A/B/C   /A/B/C

  ~ 代表的是用户的主目录

  cd  或则 cd ~ 直接切换到用户的主目录

  在Linux系统中  以“ .”开头的文件默认是隐藏文件 

  clear 清屏

  ls -a 查看目录下所有文件

  








ls -l 列表形式查看文件或目录

                 lrwxrwxrwx.   1 root root    7 Oct 18 16:23 bin -> usr/bin


      l rwx rwx rwx.  --文件的类型和权限说明

      总共十位     第一位表示的是文件的类型

                   - 表示是普通文件

                   l 表示的是链接文件

                   d 表示的是文件夹


                   其他九位分成三部分

                   前面三位表示  表示当前用户对该文件的操作权相

                   中间三位表示  表示用户组对该文件的操作权限

                   后面三位表示  表示其他用户对该文件的操作权限




      操作权限分三种 ：读取权限  写入权限   执行权限

                       r:读取    w:写入     x：执行权限  - :表示没有权限



      1            --文件或文件夹下面的数量

      root         --文件属于的用户

      root         --文件属于的用户组
 
      7            --文件的大小 byte

      Oct 18 16:23 --最近修改日期

      bin          --文件名

      -> usr/bin   --连接的真实目录
